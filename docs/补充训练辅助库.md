# 补充训练库

**1. 特征工程:**
    - tsfresh - 自动化时间序列特征提取
    - featuretools - 自动化特征工程
    - category_encoders - 分类变量编码
**2. 模型解释性:**
    - SHAP - 模型解释和特征重要性分析
    - eli5 - 模型调试和解释
**3. 实验追踪:**
    - MLflow - 实验管理、参数追踪、模型版本管理
    - Weights & Biases (wandb) - 实验可视化和协作
    - TensorBoard (已在Ptrade中) - 训练过程可视化
**4. 数据验证:**
    - great_expectations - 数据质量检查
    - pandera - pandas DataFrame 验证
**5. 交叉验证优化:**
    - scikit-optimize - 贝叶斯优化(Optuna的替代品)
    - hyperopt - 超参数优化
**6. 不平衡数据处理:**
    - imbalanced-learn - 处理不平衡数据集   

## 高优先级 (立即有用):
- MLflow/wandb - 追踪每次训练的参数和结果,对比模型性能
- SHAP - 理解模型为什么做出预测,对金融场景很重要
- imbalanced-learn - 金融数据常有类别不平衡问题

## 中优先级 (提升效率):
- tsfresh - 自动生成大量时间序列特征
- pandera - 确保输入数据格式正确

## 实用价值:
1. MLflow - 追踪每次实验的超参数、指标、模型文件,方便对比和回溯
2. SHAP - 解释模型预测,了解哪些特征影响最大(金融监管和风控很重要)
3. imbalanced-learn - 处理涨跌预测、异常检测等不平衡场景
4. tsfresh - 自动从时间序列生成几百个统计特征,省去手工特征工程
5. pandera - 验证数据格式,防止训练时数据错误

这些库都只在训练阶段使用,最终模型文件里不会包含它们的代码,所以不影响 Ptrade 兼容性。
